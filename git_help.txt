"""ГРАФИК РАБОТЫ С GIT ДЛЯ ПРОЕКТА SYSTEMIO
==========================================

1. ЕЖЕДНЕВНЫЙ ЦИКЛ (Правило: "Утром тяни, вечером толкай")
-------------------------------------------------------
Перед началом работы на любом устройстве:
   git pull origin main
   (Забирает все изменения из облака, чтобы вы работали с актуальным кодом).

После завершения работы (перед закрытием устройства):
   git add .
   git commit -m "описание того что сделал"
   git push origin main
   (Отправляет ваш код в облако, чтобы он стал доступен на остальных 5 устройствах).

2. РАБОТА С ВЕТКАМИ (Создание "песочницы")
------------------------------------------
Создать новую ветку и перейти в нее:
   git checkout -b название_ветки

Посмотреть, в какой ветке вы сейчас (отмечена звездочкой):
   git branch

Вернуться обратно в основную ветку (main):
   git checkout main

3. СИНХРОНИЗАЦИЯ VPS С НОВОЙ ВЕТКОЙ
-----------------------------------
Если вы создали ветку "admin-dev" на компе, VPS о ней не узнает сам.
Нужно зайти на VPS по SSH и ввести:
   git fetch
   (Сервер обновит список веток из облака).

   git checkout admin-dev
   (Сервер переключит файлы в папке на код из новой ветки).

4. РЕШЕНИЕ ПРОБЛЕМ (Когда Гит "ругается")
-----------------------------------------
Ошибка: "You have unstaged changes" (Вы что-то правили и не сохранили).
   Решение: Либо сделайте commit, либо, если правки не нужны и надо просто всё стереть:
   git reset --hard HEAD
   git clean -fd

Ошибка: "Diverging branches" (В облаке и на устройстве код разошелся).
   Решение (силовое): Принудительно сделать локальную папку такой же, как в облаке:
   git fetch origin
   git reset --hard origin/main

Ошибка: "fatal: The current branch has no upstream" (Первый пуш в новую ветку).
   Решение:
   git push --set-upstream origin название_ветки

5. СТАТУС И КОНТРОЛЬ
--------------------
Проверить, что изменено и что готово к отправке:
   git status

Посмотреть историю последних правок:
   git log --oneline
